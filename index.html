Pipenv

1 / 20

El Problema del Flujo Clásico

El flujo con `venv` + `pip` + `requirements.txt` funciona, pero es manual y propenso a errores.

2 / 20

¿Qué es Pipenv?

Una herramienta que combina la gestión de entornos virtuales y la gestión de paquetes en un solo comando.

3 / 20

Instalación

Pipenv se instala globalmente (o por usuario) con pip.

pip install --user pipenv
4 / 20

Iniciar un Proyecto

Navega a la carpeta de tu nuevo proyecto y simplemente instala tu primer paquete.

cd mi_proyecto
pipenv install requests

Esto crea el entorno virtual Y los archivos `Pipfile` y `Pipfile.lock`.

5 / 20

El Archivo `Pipfile`

Es el reemplazo de `requirements.txt`. Separa las dependencias de producción de las de desarrollo.

[packages]
requests = "*"

[dev-packages]
pytest = "*"
6 / 20

Dependencias de Producción

Son los paquetes que tu aplicación necesita para funcionar. Se instalan en el servidor de producción.

# Ejemplos:
# flask, requests, pandas, numpy

Se instalan con `pipenv install `

7 / 20

Dependencias de Desarrollo

Son los paquetes que solo necesitas para desarrollar la aplicación. No se instalan en producción.

# Ejemplos:
# pytest (para testing)
# black (para formateo de código)
# ruff (para linting)

Se instalan con `pipenv install --dev`

8 / 20

El Archivo `Pipfile.lock`

Este archivo bloquea las versiones exactas de TODOS los paquetes. Garantiza builds deterministas y reproducibles.

9 / 20

Builds Deterministas

Significa que si otro desarrollador instala las dependencias, obtendrá exactamente el mismo entorno, bit por bit, evitando sorpresas.

10 / 20

Activar el Entorno: `pipenv shell`

Este comando activa el entorno virtual. Es el equivalente a `source venv/bin/activate`.

pipenv shell
11 / 20

Ejecutar Comandos: `pipenv run`

Puedes ejecutar un comando dentro del entorno virtual sin necesidad de activarlo.

pipenv run python mi_script.py
pipenv run pytest
12 / 20

Instalar en un Nuevo Entorno

Para instalar todas las dependencias (producción + desarrollo) al clonar un proyecto.

git clone ...
cd mi_proyecto
pipenv install --dev
13 / 20

Instalar solo para Producción

En un servidor de producción, no necesitas las herramientas de desarrollo. Omitir `--dev` es más rápido y seguro.

pipenv install
14 / 20

Desinstalar Paquetes

Pipenv se encarga de eliminar el paquete y actualizar los archivos `Pipfile` y `Pipfile.lock`.

pipenv uninstall requests
15 / 20

Actualizar Paquetes

Para actualizar un paquete a su última versión (y actualizar el `Pipfile.lock`).

pipenv update requests
16 / 20

Ver el Gráfico de Dependencias

Un comando muy útil para visualizar el árbol de dependencias de tu proyecto.

pipenv graph
17 / 20

Alternativas Modernas

Herramientas como Poetry y PDM han ganado mucha popularidad. Siguen una filosofía similar pero usan el estándar `pyproject.toml`.

18 / 20

Conclusión

Pipenv simplifica drásticamente la gestión de dependencias, promoviendo un flujo de trabajo más robusto y reproducible al separar las dependencias de producción y desarrollo.

19 / 20

Flujo de trabajo unificado

Pipenv simplifica drásticamente la gestión de dependencias y entornos en Python, promoviendo un flujo de trabajo más robusto y reproducible. Es un gran paso adelante respecto al manejo manual con `pip` y `venv`.
